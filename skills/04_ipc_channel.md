# Skill 04 - IPC 跨进程通信（共享内存通道）

## 核心思想

消息总线在**进程内**传递消息；当需要在**不同进程**之间高效传数据时，使用基于 **POSIX 共享内存**的 IPC 通道（IPC Channel）。共享内存避免了网络栈和 socket 的拷贝开销，适合高频、低延迟场景。

## 与消息总线的对比

| 特性 | Message Bus | IPC Channel |
|------|-------------|-------------|
| 通信范围 | 进程内 | 跨进程 |
| 数据拷贝 | 一次（发布时拷贝到总线） | 零拷贝（共享内存） |
| 延迟 | 微秒级 | 亚微秒级 |
| 使用复杂度 | 低 | 中 |
| 适用场景 | 模块间解耦 | 高频数据流（图像、点云） |

## 实现原理

```
进程 A (写端)                  进程 B (读端)
     │                               │
     │  shm_open / mmap              │  shm_open / mmap
     │       │                       │       │
     │       ▼                       │       ▼
     │  ┌─────────────────────────────────────────┐
     │  │         共享内存区域 (ring buffer)        │
     └──│──▶  [head | data ... | tail]  ◀─────────┘
        └─────────────────────────────────────────┘
              sem_post ──▶  sem_wait（通知）
```

## 核心 API

```c
#include "ipc_channel.h"

/* === 写端（发布进程）=== */

// 创建通道（第一次调用创建共享内存）
IpcChannel* ch = ipc_channel_create("my_channel",
                                    sizeof(SensorMsg),  // 每条消息大小
                                    64);                // 队列深度

// 写入消息（非阻塞）
SensorMsg msg = { .lat = 39.9, .lon = 116.4 };
ipc_channel_write(ch, &msg);

/* === 读端（消费进程）=== */

// 打开已存在的通道
IpcChannel* ch = ipc_channel_open("my_channel");

// 阻塞读取（有消息才返回）
SensorMsg msg;
ipc_channel_read(ch, &msg);
printf("lat=%.4f\n", msg.lat);

/* === 关闭 === */
ipc_channel_close(ch);   // 写端或读端关闭自己的映射
ipc_channel_unlink("my_channel");  // 最后一个进程删除共享内存
```

## 环形缓冲区设计

共享内存区域通常实现为**无锁环形缓冲区**（lock-free ring buffer），写端和读端各维护自己的指针，通过原子操作保证并发安全：

```
┌────────────────────────────────────────────────┐
│ magic | msg_size | capacity | head | tail | ... │  ← 控制块（在共享内存开头）
├────────────────────────────────────────────────┤
│  [slot 0] [slot 1] [slot 2] ... [slot N-1]     │  ← 数据区
└────────────────────────────────────────────────┘
  head（写端递增）   tail（读端递增）
```

## 同步机制

- **写端通知读端**：`sem_post` POSIX 信号量
- **读端等待**：`sem_wait` 阻塞，无消息时不占 CPU
- **溢出策略**：队列满时，写端可选择丢弃最旧数据（覆盖）或阻塞等待

## 注意事项

1. **消息大小固定** — 共享内存通道每个 slot 大小在创建时确定，不支持变长消息。对变长数据可传递指针索引，数据本身放另一块共享内存。
2. **进程崩溃** — 若写端崩溃，共享内存不会自动删除，需在启动时处理残留。
3. **命名冲突** — 通道名最终对应 `/dev/shm/<name>`，不同服务应使用唯一名称。
4. **权限** — 多用户场景需注意 `shm_open` 的 `mode` 参数。

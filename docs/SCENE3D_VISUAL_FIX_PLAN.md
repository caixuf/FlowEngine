## 1. 参考目标

下图为用户提供的理想路面/街景参考：

![reference-road-surface](reference_road_surface.png)

**从图中提取的核心视觉特征：**

| 元素 | 特征 |
|------|------|
| **路面** | 浅灰色沥青，平整但带细微颗粒感；不追求真实 PBR 车漆级反射，走“干净卡通”路线 |
| **车道线** | 左侧白色虚线，中间黄色双实线（分隔带），线宽适中、边缘清晰 |
| **路肩** | 深色土路/路肩，与沥青路面有明显明暗分界 |
| **护栏** | 黑色金属护栏沿道路两侧连续分布 |
| **植被** | 草地（低饱和绿色），远处锥形/三角形树木（深绿），排列有节奏感 |
| **远景** | 白色/浅蓝低多边形山脉，蓝色渐变天空，少量扁平白云 |
| **光照** | 明亮日光，阴影清晰但不生硬；车辆有底盘接触阴影 |
| **整体风格** | 低多边形 + 明亮卡通，远处细节简化，近处可读性强 |

**设计原则**：
- 不追真实照片级渲染，先达到“干净、明亮、一眼能看清车道和障碍物”的卡通风格。
- 优先解决“路面像塑料/车道线糊/路灯怪/街景空”的问题。
- 所有改动必须保证 `demo_evaluator.py` CI 回归不退化。

## 4. 详细方案

### A1 沥青路面材质

**现状问题**
- 当前程序化 canvas 纹理色块+颗粒+裂缝，但颜色偏深（0x2a2d30），整体显脏。
- 参考图为浅灰、干净、低对比度路面。

**方案**
- 新建 `_makeAsphaltTextureV2()`：
  - 底色改为 `#7a7d80`（浅灰），随机噪点 `#6a6d70` / `#8a8d90` 叠加，模拟沥青颗粒。
  - 去掉明显裂缝，避免显旧。
  - roughness 0.85–0.95，metalness 0.0。
- 路面几何保持 ribbon mesh，vertex color 做轻微脏迹渐变。
- 可选：在性能档位 `low` 时回退到纯色 MeshBasicMaterial。

**涉及文件**
- `tools/flowboard/js/scene3d/view/RoadView.js` 或 `scene3d.js` 中路面构建段

---

### A2 车道线系统

**现状问题**
- 已改为 ribbon mesh，但颜色/线型单一，未区分白虚线/黄实线。

**方案**
- 按车道数构建标线：
  - 最左侧车道左边缘：白色实线（路缘线）。
  - 车道之间：白色虚线（6m 线 + 9m 空，或统一 4m 线 + 6m 空）。
  - 对向分隔：黄色双实线（参考图中间分隔带）。
- 材质用 `MeshBasicMaterial` + 少量 emissive（0.2），确保暗光下可见。
- 标线厚度 0.12–0.15m，宽度 0.15m（虚线）/ 0.20m（实线）。

**涉及文件**
- `tools/flowboard/js/scene3d/view/RoadView.js`

---

### A3 路肩与护栏

**现状问题**
- 当前路肩颜色与路面接近，没有护栏，道路边界感弱。

**方案**
- 路肩：在沥青外侧加一条 0.5–0.8m 宽的土路/碎石带，颜色 `#5a4d40`（深褐灰）。
- 护栏：沿道路两侧每 4m 一个黑色金属立柱 + 两横杆，用 `InstancedMesh` 合并 draw call。
- 护栏只在直线/大曲率路段放置，急弯匝道可省略或降低密度。

**涉及文件**
- `tools/flowboard/js/scene3d/view/RoadView.js`

---

### A4 车辆底盘阴影

**现状问题**
- 当前车辆没有阴影，视觉上“飘”。

**方案**
- 每个车辆下方加一个透明黑色 Plane（或圆形 Decal），尺寸略大于车辆投影：
  - 轿车：`2.2m × 5.0m` 椭圆，透明度 0.35。
  - 行人：小圆斑。
- 阴影跟随车辆位置，但不跟随车身 roll/pitch（只取水平投影）。
- 用 `MeshBasicMaterial({ transparent: true, opacity: 0.35, color: 0x000000 })`。
- 在 `low` 档位关闭阴影以省 draw call。

**涉及文件**
- `tools/flowboard/js/scene3d/view/VehicleView.js` 或 `scene3d.js` 车辆更新循环

---

### B1 草地与树木

**现状问题**
- 当前环境物体稀疏，街景空。

**方案**
- 草地：道路两侧各 50–100m 范围铺大片绿色平面，颜色 `#6a9e5a`，加轻微噪声纹理。
- 树木：
  - 低多边形锥形树（圆柱树干 + 圆锥树冠），树干棕、树冠深绿。
  - 按距离分布：近处 5–15m 一棵，远处逐渐减少密度。
  - 使用 `InstancedMesh` 合并，限制总数 200–500（按性能档位）。
- 城市路段可穿插少量建筑方块（简化为色块，不细节）。

**涉及文件**
- `tools/flowboard/js/scene3d/view/EnvironmentView.js`（新增）

---

### B2 远山与天空

**现状问题**
- 当前天空是纯色或简单渐变，没有远景层次。

**方案**
- 天空：蓝色渐变球/半球，顶部 `#4a90e2`、底部 `#a8d0f0`。
- 云：少量扁平白色椭圆片，缓慢漂移（可选动画）。
- 远山：在场景边界（x≈-500~2500，z≈±400）放置低多边形山脉环，白色/浅蓝 `#c8d8e8`，用雾（fog）淡化远景。
- 雾：启用 `THREE.Fog`，颜色与天空底部一致，距离 200m–800m，增强纵深。

**涉及文件**
- `tools/flowboard/js/scene3d/view/EnvironmentView.js`（新增）
- `tools/flowboard/js/scene3d.js` 渲染器初始化

---

### B3 路灯重做

**现状问题**
- 当前路灯是发光方块，形状奇怪。

**方案**
- 替换为杆+灯罩模型：
  - 灰色金属杆，高 8m，立于路肩外侧。
  - 顶部 T 形或悬臂灯罩，朝下。
  - 灯罩用 emissive 材质模拟发光，但不加真实 PointLight（性能考虑）。
- 路灯密度：城市道路每 20m 一盏，高速/匝道稀疏或省略。
- 用 `InstancedMesh` 合并。

**涉及文件**
- `tools/flowboard/js/scene3d/view/RoadView.js` 或 `EnvironmentView.js`

---

### C1 碰撞视觉反馈（可选）

**方案**
- 后端：`monitor_node.c` 订阅 `sim/collision`，把最近一次事件（时间戳+位置）透传到 `metrics.scene.collision_event`。
- 前端：在碰撞位置触发 0.5s 闪光 sprite（半透明 expanding ring）+ 屏幕轻微震动（camera shake）。
- 事件 2s 后自动清除。

**涉及文件**
- `modules/adas_nodes/monitor_node.c`
- `tools/flowboard/js/scene3d.js`

## 5. 数据契约与后端配合

本次视觉升级主要改前端，但以下字段/约定需要前后端保持一致：

- `metrics.scene.road_network`：已支持多段道路，每段含 `nodes` / `lanes` / `lane_width`，前端据此构建路面/标线/护栏。
- `metrics.scene.entities`：已透传全部 NPC + TL + ETC + stop_line，前端按 `type` 渲染。
- `sim/collision`：如做 C1 碰撞反馈，需要 monitor 透传事件。

详见 [FLOWBOARD_SCENE_CONTRACT.md](FLOWBOARD_SCENE_CONTRACT.md)。

## 7. 执行顺序建议

按 **A1 → A2 → A3 → A4 → B1 → B2 → B3 → C1** 推进：

1. 先让路面和标线接近参考图（视觉冲击最大）。
2. 加路肩、护栏、阴影，让车辆“落地”。
3. 补环境和远景，解决街景空的问题。
4. 最后做碰撞反馈等交互细节。
